## はじめに

G検定に合格したので、勉強中に感じたことや、やってよかったことをまとめます。

この記事は、すごく綺麗な学習ロードマップというより、どちらかというと「実際に勉強してみたら、ここで詰まった」「でもここが分かると急に景色が変わった」という体験記です。

これからG検定を受ける方、とくに勉強を始めたばかりで、

- 用語が多すぎてつらい
- 機械学習とディープラーニングの違いがふわっとしている
- CNN、RNNあたりで頭がこんがらがる
- 結局、合格目的なら何を優先すればいいの？

という方の参考になればうれしいです。

## 最初にぶつかった壁: とにかく用語が多い

G検定の勉強を始めて最初に思ったのは、シンプルに「用語、多くない？」でした。

AI、機械学習、ディープラーニング、教師あり学習、教師なし学習、強化学習、過学習、正則化、CNN、RNN、LSTM、Attention、Transformer……。

最初は、知らない単語が順番待ちしている感じでした。しかも、それぞれが完全に別物というより、少しずつ関係しているので余計にややこしい。

この段階では、ひとつひとつを完璧に覚えようとするより、まずは大きな棚を作るのが大事だと感じました。

たとえば、私はこんな感じで整理しました。

| 分類 | ざっくりした意味 |
| --- | --- |
| AI | 人間っぽい知的な処理をコンピュータにさせる大きな考え方 |
| 機械学習 | データからルールやパターンを学ぶ方法 |
| ディープラーニング | ニューラルネットワークを深くした機械学習の一種 |
| 教師あり学習 | 正解付きデータで学ぶ |
| 教師なし学習 | 正解なしデータから構造を見つける |
| 強化学習 | 行動して報酬をもらいながら学ぶ |

この棚ができると、用語がただの単語の群れではなく、「どの箱に入る話なのか」で見えるようになりました。

## 機械学習の概念が分かると、少し楽になる

用語に苦戦したあと、少し楽になったのは「機械学習って、結局はデータからパターンを見つける話なんだ」と腑に落ちてからです。

たとえば、メールがスパムかどうかを分類する場合、人間がすべてのルールを手書きするのではなく、過去のメールデータから特徴を学ばせる。  
住宅価格を予測するなら、過去の価格、広さ、駅からの距離などから関係性を学ばせる。

このあたりを理解すると、教師あり学習・教師なし学習・強化学習も、急に少しだけ人間味が出てきます。

- 教師あり学習: 答え付きドリル
- 教師なし学習: 似ているものを自分でグループ分けする自由研究
- 強化学習: うまくいったらポイントが入るゲーム

雑に見えるかもしれませんが、最初はこれくらいのイメージで十分でした。

G検定は範囲が広いので、最初から全部を厳密に理解しようとすると、かなり体力を持っていかれます。まずは「これは何の話か」を掴む方が、後から細かい知識を乗せやすかったです。

特に、教師あり学習・教師なし学習に出てくる各アルゴリズムを、序盤から深く理解しようとすると沼にハマりやすいです。

線形回帰、ロジスティック回帰、SVM、決定木、ランダムフォレスト、クラスタリング、主成分分析……と追いかけ始めると、気づいたら「G検定の勉強」ではなく「機械学習の各論」に潜っていきます。

ただし、試験で問われることもあるので、やらなすぎるのも危ないです。  
私の場合は、まず「何に使うアルゴリズムなのか」「教師あり・教師なしのどちら側なのか」「似た用語と何が違うのか」を優先して押さえるようにしました。

深掘りは大事。でも、最初から全員と親友になろうとすると時間が溶けます。まずは顔と名前と所属部署を覚える、くらいの距離感がちょうどよかったです。

## なぜかピンときたCNNとRNN

勉強していて、個人的に「お、これは少し分かるかも」と思ったのがCNNとRNNでした。

CNNは画像認識でよく使われるモデルです。  
画像を小さな領域ごとに見ながら、線や形などの特徴を取り出していくイメージです。

最初は難しく見えますが、考え方としては「画像の中から特徴を探す虫眼鏡」みたいなものだと捉えると、少し分かりやすくなりました。

一方、RNNは時系列データや文章のように、順番が大事なデータを扱うモデルです。

たとえば文章では、単語が単体で存在しているだけではなく、前後の流れで意味が変わります。  
「今日は雨が」と来たら、次に「降る」「降った」などが来そうだと予想できますよね。

RNNは、前の情報を次に引き継ぎながら処理するイメージです。

もちろん、実際には勾配消失問題やLSTM、GRUなどの話も出てきます。  
ただ、最初から全部を一気に理解しようとせず、

- CNN: 画像の特徴を見つけるのが得意
- RNN: 順番のあるデータを扱うのが得意

と押さえるだけでも、問題文を読むときの迷子感が減りました。

今思うと、CNNは「画像」、RNNは「自然言語処理や時系列データ」みたいに、ざっくり対応する用途で覚えられたのが、とっつきやすかった理由かもしれません。

もちろん、厳密にはCNNが画像だけ、RNNが自然言語処理だけというわけではありません。  
でも試験勉強の入口としては、「このモデルは何が得意なのか」をラベルとして貼っておくと、かなり楽になります。

## 合格目的なら、ぶっちゃけ過去問中心でもかなり戦える

ここは正直に書きます。

G検定に合格することだけを目的にするなら、過去問や問題演習を中心に進めるのはかなり有効だと思いました。

理由は、G検定では「知っているかどうか」で解ける問題が多いからです。  
もちろん、概念理解は大事です。ただ、試験本番では限られた時間で多くの問題を処理する必要があります。

なので、私の場合は、

1. まず全体像をざっと見る
2. 問題を解く
3. 間違えた用語を調べる
4. 似た用語との差分をメモする
5. また問題を解く

という流れがしっくりきました。

教科書を最初から最後まで完璧に読むというより、問題演習で穴を見つけて埋めていく感じです。

ただし、過去問だけに寄せすぎると、「受かったけど説明できない」状態になりやすいです。  
合格後もAIを使ったり、発信したり、仕事に活かしたりしたいなら、最低限の概念理解はやっぱり大事だと感じました。

## やってよかったこと

### 1. 用語を自分の言葉で言い換える

公式テキストの説明をそのまま覚えようとすると、私はけっこう苦しくなりました。

なので、たとえば次のように自分の言葉に変えていました。

- 過学習: 練習問題だけ異常に得意になって、本番に弱い状態
- 正則化: 覚えすぎを防ぐためのブレーキ
- 教師あり学習: 答えを見ながら学ぶ
- 教師なし学習: データの中の似たもの探し

正確さを失わない範囲で、まず自分が思い出せる言葉にする。  
これがかなり効きました。

### 2. 似ている用語をセットで覚える

G検定では、似た用語がたくさん出てきます。

- 教師あり学習 / 教師なし学習 / 強化学習
- CNN / RNN
- LSTM / GRU
- 正則化 / ドロップアウト / バッチ正規化
- Precision / Recall / F値

単語を単体で覚えるより、「何が違うのか」をセットで見る方が記憶に残りやすかったです。

### 3. 問題演習で自分の弱点を見つける

読んで分かった気になっていても、問題を解くと普通に間違えます。なかなか正直な世界です。

でも、間違えたところは伸びしろです。

私は、間違えた問題を見て、

- 用語を知らなかったのか
- 似た用語と混同したのか
- 問題文の読み方を間違えたのか

を軽く分けるようにしていました。

これをやると、ただ解くだけよりも復習の効率が上がります。

## 本番で感じたこと

試験本番は、90分で145問でした。

受験前は、ざっくり「60分で一通り解いて、残り30分で見直そう」と考えていました。  
ただ、実際には思ったより時間がかかり、見直しに使えたのは残り10分程度でした。

ここで反省したのが、見直したい問題にチェックを付けすぎたことです。

不安な問題にどんどんチェックを付けていたら、最後に見直すときに「どれが本当に自信のない問題だったっけ？」となってしまいました。

チェックを付けること自体は大事です。  
ただ、全部に近い勢いで付けると、チェックの意味が薄くなります。

次に受ける人には、

- まったく分からない問題
- 2択まで絞ったけど迷う問題
- 計算や読み間違いがありそうな問題

のように、チェックの基準を決めておくのをおすすめしたいです。

本番は知識だけでなく、時間配分と見直し戦略もかなり大事でした。

## 反省点

反省点は、序盤に用語を丁寧に理解しようとしすぎて、少し手が止まったことです。

もちろん理解は大切ですが、G検定は範囲が広いので、序盤から完璧主義になると進みません。

最初は、

- 分からない用語があっても先に進む
- 問題演習で何度も出るものを優先する
- 重要そうなところから深掘りする

くらいの割り切りでよかったなと思います。

勉強中は、分からない用語に出会うたびに足を止めたくなります。  
でも、全部に立ち止まると、知識の渋滞が起きます。頭の中で用語たちが交差点に集まりすぎる感じです。

まずは交通整理。あとから細かく理解する。  
この順番の方が、自分には合っていました。

## これから受ける人へのアドバイス

これからG検定を受ける方には、次の流れをおすすめしたいです。

1. まずAI・機械学習・ディープラーニングの関係をざっくり掴む
2. 教師あり・教師なし・強化学習を整理する
3. CNN、RNN、Transformerなど代表モデルの用途を押さえる
4. 問題演習を早めに始める
5. 間違えた用語だけを自分の言葉でメモする
6. 見直しチェックの基準を決めておく
7. 試験直前は頻出用語と法律・倫理・社会実装まわりを確認する

特に、合格目的なら問題演習は早めでいいと思います。

「まだ理解していないから問題を解けない」ではなく、「問題を解きながら理解する」くらいでちょうどよかったです。

## まとめ

G検定の勉強は、序盤の用語ラッシュがなかなか大変でした。

ただ、AI・機械学習・ディープラーニングの関係が見えてくると、少しずつ整理できるようになります。  
CNNやRNNのような代表的なモデルも、「何が得意なのか」から入るとかなり理解しやすかったです。

合格だけを目指すなら、過去問や問題演習中心でもかなり戦えます。  
ただ、せっかく勉強するなら、用語を自分の言葉で説明できるところまで持っていくと、合格後にも知識が残りやすいです。

本番では、90分145問という時間の中で、思ったより余裕がありませんでした。  
見直しチェックを付ける基準も、事前に決めておくと安心です。

これから受ける方は、最初から完璧を目指さず、まずは全体像をつかんで、問題演習で少しずつ穴を埋めていくのがおすすめです。

合格までの道のり、用語は多いですが、ひとつずつ整理すればちゃんと進めます。
